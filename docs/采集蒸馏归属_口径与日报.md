# 采集 / 蒸馏 / 归属 — 口径说明与日报统计

**用途**：统一「今日采集多少、蒸馏多少、归属多少」的统计口径，避免多源、多 id 导致数字不一致。

---

## 一、三条数字的权威来源与定义

| 指标 | 定义 | 权威数据来源 | 如何统计 |
|------|------|--------------|----------|
| **今日新采集** | 当天由采集脚本**新写入** 待蒸馏/ 的条目数 | `distill_input_history.json` | 统计 `source_type == "feed_entry"` 且 `first_seen_at` 为当天的条目数；或统计当天写入 待蒸馏/ 的文件数。 |
| **今日已蒸馏** | 当天**完成蒸馏**的条目数（产出已写入 蒸馏产出_待归属） | `distill_input_history.json` | 统计 `distilled_at` 为当天的条目数。 |
| **今日已归属** | 当天从 蒸馏产出_待归属 **迁移到** about_* 等知识库的条数 | `docs/蒸馏归属目标.md` | 看当天「本次迁移记录」节的表格行数或文末合计。 |

**同一条内容在 history 里可能出现两次**：采集时（feed_entry，id = URL）；蒸馏时（document，id = 文件路径）。「今日新采集」只按 **feed_entry + first_seen_at 当天** 计。

---

## 二、每日怎么查（推荐步骤）

1. **今日新采集**：打开 `distill_input_history.json`，搜 `"first_seen_at": "YYYY-MM-DD"`，只数 `"source_type": "feed_entry"` 的条数；或看当天采集脚本输出「已写入 N 条待蒸馏」。
2. **今日已蒸馏**：搜 `"distilled_at": "YYYY-MM-DD"`，统计出现次数。
3. **今日已归属**：打开 `docs/蒸馏归属目标.md`，找到当天日期的「本次迁移记录」节，看表格行数或「合计」行。
4. **可选**：在 `scripts/` 下加 `stats_distill_daily.py`，读 history + 归属目标文档，按日期输出三列数字。

---

## 三、流程与口径关系（简要）

```
多源采集 → 待蒸馏/（+ history）
    ↓
distill_run（单条或 --target 批量）→ 蒸馏产出_待归属/（+ history 记 distilled_at）
    ↓
assign_and_migrate（按 归属匹配规则.json）或 migrate_by_csv → about_*（+ 蒸馏归属目标.md 本次迁移记录）
```

- **采集**：以「新写入 待蒸馏」或「history 里当天 feed_entry」为准。  
- **蒸馏**：以「history 里当天 distilled_at」为准。  
- **归属**：以「蒸馏归属目标.md 里当天迁移记录」为准。  

**流程总览**：见 **docs/流程说明.md**。
