# 自动化流水线结果导入本地

当 Cursor automation 在远端/容器内执行采集 → 蒸馏 → 迁移（如 20260311 批次，commit 0dda302）并 push 后，本地需做两类同步。

---

## 一、项目文件同步（knowledge_distill_agent）

保留本地未提交修改并拉齐到 automation 推送状态：

```bash
cd knowledge_distill_agent
bash scripts/sync_from_automation.sh
```

脚本会：`git stash` → `git fetch origin` → `git reset --hard origin/main`。

**若同步后只有 20 条（如 20260307_distill_001～020、commit d71fa13）**：说明当前 `origin/main` 尚未包含 20260311 批次（0dda302）。请在本机执行：

```bash
cd knowledge_distill_agent
git fetch origin
git log origin/main -5 --oneline
```

- 若列表里已有 `0dda302 chore: run automation pipeline for 20260311 batch`，再执行一次 `bash scripts/sync_from_automation.sh`（或 `git reset --hard origin/main`）即可拿到 50 条批次。
- **若没有 0dda302**：常见原因是 automation 把提交推到了 **Cursor 自动分支**（如 `cursor/-bc-4877716e-...`）而不是 `main`。此时：
  - **方式 A（推荐）**：在 **automation 的 workspace** 里执行，把该分支推成 main：
    ```bash
    git push origin cursor/-bc-4877716e-06d3-4c82-a08c-9702ccb1caa5-eef3:main
    ```
    然后本机执行 `git fetch origin && git reset --hard origin/main` 即可拿到 50 条。
  - **方式 B**：本机不改远程 main，只合并该分支：
    ```bash
    git fetch origin
    git merge origin/cursor/-bc-4877716e-06d3-4c82-a08c-9702ccb1caa5-eef3 -m "merge: 20260311 automation batch"
    ```
  - **或从 automation 导出**：从 automation 工作区拷贝 `迁移映射_20260311.csv` 及 50 个已迁移 .md，按下文「二、迁移结果导入」方式 A/B 落盘。

若本机显示 `Your branch is behind 'origin/main' by 1 commit`，说明远程 main 已是 20260311 批次，直接执行：

```bash
git pull
```

即可快进到 0dda302（50 条）。

同步成功后你将得到：

- 迁移映射 `迁移映射_20260311.csv`（在项目根或 `知识抽取和蒸馏/`，视 automation 落盘位置）
- 更新后的 `待蒸馏/`（含新增 77 条）
- `蒸馏产出_待归属/` 中 20260311 批次已迁出（剩余 0）
- 与远端一致的脚本、config、docs

恢复之前本地修改（若需要）：`git stash list` → `git stash pop`。

## 二、迁移结果（50 条 .md）导入本地知识库

Automation 使用 **KB_VENTURE=/workspace**，迁移写入了 **/workspace/about_***，你本地知识库是 **~/kb_venture**（VentureSights培训）。要让这 50 条出现在本地 about_*，任选其一即可。

### 方式 A：从 automation 工作区拷贝（推荐，若可访问）

若你能访问 automation 运行后的工作区（例如 Cursor 提供的 workspace 或 CI 产物）：

1. 从该环境的 `/workspace/about_industry_analysis/所有产业链框架汇总/`、`/workspace/about_startup/` 等中，找出本次新增的、映射表里列出的 50 个 .md 文件。
2. 按相同相对路径拷贝到本地：
   - `~/kb_venture/about_industry_analysis/所有产业链框架汇总/`
   - `~/kb_venture/about_startup/`
   - 等（以 `迁移映射_20260311.csv` 中的 target_key 与 target_basename 为准）。

或直接打包 automation 的 `about_*` 下本次新增文件，在本地解压到 `~/kb_venture/`。

### 方式 B：用同一映射在本地再执行一次迁移

若你手上有 **50 个源文件**（20260311_distill_001.md～050.md，即迁移前的蒸馏产出），可在本地用同一张 CSV 再迁一次，落盘到本地知识库：

```bash
cd knowledge_distill_agent
export KB_VENTURE=~/kb_venture
# 将 50 个源文件（20260311_distill_001.md～050.md）放回 蒸馏产出_待归属/
KB_VENTURE=~/kb_venture python3 scripts/migrate_by_csv.py 迁移映射_20260311.csv
```

CSV 格式为 `source_basename, target_key, target_basename`；`migrate_by_csv.py` 从项目下的 `蒸馏产出_待归属/` 读源文件，写入 `$KB_VENTURE/about_*/`。  
若 automation 未保留这 50 个源文件，需从 artifact 或备份恢复后再执行。

### 方式 C：知识库本身即 git 仓库且 automation 已 push

若 `/workspace` 对应的是与 `~/kb_venture` 同一远程仓库（例如 VentureSights培训 的 clone），且 automation 在迁移后对知识库做了 commit + push，则本地只需在知识库目录执行：

```bash
cd ~/kb_venture
git pull
```

即可拿到本次迁移新增的 50 个 .md。

## 五、从 GitHub「蒸馏产出_待归属」同步到本地并迁入 $KB（现成路径）

新蒸馏产出在 GitHub 的 **cursor 分支** 的「蒸馏产出_待归属」文件夹里，各分支可能含不同日期批次（如 20260307、20260311）。本地需两步：先拉取该文件夹，再按规则迁入 `$KB/about_industry_analysis/所有产业链框架汇总` 等**现成路径**（不新建路径）。

### 1. 拉取 蒸馏产出_待归属

在项目根执行（先 `git fetch origin` 若需最新分支）：

```bash
cd knowledge_distill_agent
git fetch origin   # 可选，拉齐远程分支
bash scripts/import_pending_from_branches.sh
```

脚本默认从含 **20260311** 等批次的分支导入；可指定分支：

```bash
bash scripts/import_pending_from_branches.sh "origin/cursor/-bc-3d712ded-01c1-4b83-9916-649bb37875dc-8bea" "origin/cursor/-bc-bef9f1e5-f83e-4873-b2a8-1faea95df643-f605"
```

### 2. 归属并迁移到 $KB（现成路径）

迁移目标为已有目录（如 `$KB/about_industry_analysis/所有产业链框架汇总`），只追加文件：

```bash
export KB_VENTURE=~/kb_venture   # 或你的知识库根
conda activate ikb_env
python scripts/assign_and_migrate.py --dry-run   # 先看映射
python scripts/assign_and_migrate.py --migrate    # 执行迁移
```

### 3. 若没有「今天」的批次

若 GitHub 上各 cursor 分支里目前**没有当天日期**（如 20260312）的蒸馏产出，说明当天蒸馏可能尚未推送到 GitHub。需要在跑蒸馏的环境（如 Cursor / CI）里把「蒸馏产出_待归属」推到某分支或 main，再在本机执行上述 1、2 步。
