# 信息源蒸馏与知识+任务体系更新 — 架构说明

**版本**：v1.3  
**目的**：从每日新增信息源中蒸馏出有用知识，并系统化更新知识库与任务管理体系。  
**运行**：适用于线上/云环境；所有路径由环境变量配置。信息源统一维护在 `config/信息源列表.md`（每行一条）。

---

## 一、目标与原则

- **目标**：把「每天新增的信息源」（RSS、每日新闻、产业神经联动网等）中可沉淀的内容，**蒸馏成结构化知识**，写入约定知识库；并根据蒸馏结果**产出对任务体系的更新建议**（新场景、新触发词、新知识库路径等），经维护流程纳入 COORDINATION / SCENARIO_TASKS / 维护清单。
- **原则**：单一事实来源（信息源配置、知识归属、任务体系更新入口只在约定位置）；与现有「做一次知识+任务维护」清单衔接；蒸馏产出可追溯（来源、日期、主题）。

---

## 二、数据流总览

```
信息源（每日新闻 / 知识库源 / 产业日报 / 新增 RSS）
    ↓
采集（拉取新条目）
    ↓
粗筛（区分「仅新闻」vs「可沉淀知识」）
    ↓
蒸馏（从可沉淀内容中提炼：原则、框架、案例、指标、关系等）
    ↓
知识入库（按主题写入 about_* 或约定目录）+ 更新 README/索引
    ↓
任务体系更新建议（若需新场景/新触发/新路径 → 写入建议清单，由维护流程采纳）
```

---

## 三、信息源（输入）

信息源**统一登记**在 `config/信息源列表.md`，**每行一个** URL，不区分类型。

- **RSS/Atom 订阅 URL**：拉取 feed，解析近期条目写入待蒸馏（可从 OPML 导出 xmlUrl 粘贴到列表）。
- **主页面 URL**：抓取主站及子页正文写入待蒸馏。

---

## 四、采集与粗筛

- **采集**：由 `collect_sources.py` 读取信息源列表中的 URL（RSS/Atom、主页面），拉取并写入待蒸馏。
- **粗筛**：从流中筛出**可沉淀**条目。标准：常青、有体系、可引用、深度技术/商业/产业/政策。
- **输出**：待蒸馏列表（原文或摘要 + 元数据）。
- **数量下限**：每次采集+蒸馏至少产出 10 条知识单元写入 蒸馏产出_待归属；不足时继续选条蒸馏直至达到，或说明原因。

---

## 五、蒸馏（有用知识提炼）

- **输入**：粗筛后的单条或批量内容（正文/摘要 + 来源、日期）；**非 PDF、非研报结构**（RSS、日报、博客等文本）。
- **执行**：由主 Agent 或蒸馏脚本执行；不沿用 pdf_parser 的研报蒸馏逻辑。
- **操作**：用 LLM 从内容中提炼：**原则/框架**、**案例/事实**、**指标/关系**、**术语/定义**。
- **输出**：结构化知识单元（Markdown），字段：标题、类型（原则/框架/案例/指标/术语）、主题标签、来源 URL、日期、正文。
- **归属**：按主题归入 about_*（见 `docs/蒸馏归属目标.md`）。

---

## 六、知识入库与路径约定

- **归属目标**：知识库根下 about_startup/、about_financials/、about_industry_analysis/、about_energy_storage/、about_marketing/ 等（知识库根由环境变量指定）；归属目标以 `蒸馏归属目标.md` 为准。
- **命名与索引**：新文件建议带日期与主题（如 `YYYYMMDD_主题_简短描述.md`）；对应目录下 README 或索引文件需更新。

---

## 七、任务体系更新建议

- **触发**：蒸馏或入库过程中发现：某类知识反复出现适合作为新场景；需要新触发词或新 subagent 知识库路径；维护清单需新增检查项。
- **产出**：简短「任务体系更新建议」条目，包含：建议类型、理由、建议修改位置。
- **落地**：在「做一次知识+任务维护」时，由 coordinator 产出的清单中一并查看并采纳。

---

## 八、节奏与执行责任

| 节奏 | 动作 | 执行 |
|------|------|------|
| **每日** | 采集新条目；可选：粗筛出「待蒸馏」列表 | 现有 industry_news / RSS / 日报管线 |
| **每周或按需** | 对「待蒸馏」执行蒸馏；知识入库；更新 README/索引 | 主 Agent 或人工 |
| **每月或「做一次知识+任务维护」** | 执行 coordinator「本次建议执行的清单」；处理「蒸馏-任务体系更新建议」 | coordinator 产出清单 → 主 Agent 或人工执行 |
